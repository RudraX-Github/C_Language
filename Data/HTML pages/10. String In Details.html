The Integer Illusion

1. Character Details & ASCII

In C, the char data type takes exactly 1 Byte (8 bits) of memory. But computers don't understand letters; they only understand numbers. C uses the ASCII (American Standard Code for Information Interchange) table to map integers to characters.

This means a char is actually just a small integer! You can perform math directly on characters.

char letter = 'A';

printf("%c", letter); A
// 2. Print as Integer (%d)

// 3. Character Math
char next = letter + 2; // 65 + 2 = 67
printf("%c", next); C
The Vital Terminator

2. String Creation & The NULL Terminator (\0)

In C, there is no dedicated "String" data type. A string is simply a 1D array of characters. But how does C know where the string ends if the array is larger than the word? It uses a special hidden character called the Null Terminator (\0, ASCII value 0).

Declaration Methods
// Method 1: String Literal (Auto adds \0)
char str1[10] = "Hello";
// Method 2: Character Array (Must manually add \0)
char str2[10] = {'H', 'i', '\0'};
char str[10] = " ";

3. String I/O: The Space Bar Problem

When reading strings from the user, scanf("%s", str); is dangerous because it stops reading as soon as it hits a whitespace. To safely read a full sentence with spaces, C programmers use fgets().

string_input.c
char name[10];
printf("Enter name: ");
scanf("%s", name);
fgets(name, sizeof(name), stdin);
printf("Stored: %s\\n", name);
