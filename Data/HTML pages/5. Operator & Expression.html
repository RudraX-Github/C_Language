1. Types of Operators

Operators are special symbols used to perform operations on operands (variables and values). C has a rich set of built-in operators. A critical C-specific rule to remember: Integer Division (/) truncates the decimal. 5 / 2 is 2, not 2.5.

2. Operator Precedence

When an expression has multiple operators, C needs rules to decide which operation to perform first. This is called Precedence (like BODMAS/PEMDAS in math). If operators have the same precedence, Associativity determines the direction (Left-to-Right or Right-to-Left).

Core Rules

  • *, /, % run before +, -
  • Arithmetic operators associate Left-to-Right.
  • Assignment operators (=) associate Right-to-Left.

3. Type Conversion (Casting)

What happens when you mix different data types (like adding an int to a float)? C performs Type Conversion. There are two types: Implicit (Automatic Promotion by the compiler to prevent data loss) and Explicit (Forced casting by the programmer, which can lead to data loss).

Implicit Promotion

When adding an int and a float, C automatically "promotes" the integer to a float behind the scenes to avoid losing precision. So 5 becomes 5.0, and the result is a float.
